Temas de esta guía

Cubriremos 3 pilares fundamentales: organizar archivos externos, conocer los tipos de campos y sus propiedades, y validar con JavaScript.

01

Agregar vistas desde otro archivo HTML

Vamos a crear un archivo formularioRegistro.html con un formulario completo, y luego lo mostraremos dentro de nuestro index.html como una nueva pestaña del menú. Esto permite dividir el contenido en archivos separados y cargarlo dinámicamente.

El proceso tiene 4 partes:

  • Parte A: Crear el archivo formularioRegistro.html con su propio contenido.
  • Parte B: Agregar un nuevo botón en el menú de index.html.
  • Parte C: Agregar una nueva sección con un <iframe> que cargue el archivo.
  • Parte D: El JavaScript existente ya maneja la navegación automáticamente.

Parte A — Crear el archivo formularioRegistro.html

Creamos un nuevo archivo HTML independiente en la misma carpeta del proyecto. Este archivo tiene su propio <html>, <head> y <body> completo.

ESTRUCTURA DE CARPETAS 
mi-proyecto/
├── index.html                ← página principal con el menú
├── formularioRegistro.html   ← NUEVO: formulario en archivo separado
├── style/
│   └── style.css             ← estilos CSS
└── app.js                    ← lógica de las tabs
💡 El archivo formularioRegistro.html es una página HTML completa e independiente. Puede abrirse solo en el navegador o cargarse dentro de otro HTML usando un <iframe>.

Parte B — Agregar el nuevo botón en el menú

En el archivo index.html, agregamos un tercer botón dentro del <nav>. El atributo data-tab debe tener un nombre único que usaremos como id de la sección.

HTML — index.html (menú) 
<!-- ① MENÚ DE NAVEGACIÓN -->
<nav class="menu">
  <button class="tab-btn active"
          data-tab="registro">
    Registro
  </button>
  <button class="tab-btn"
          data-tab="contacto">
    Contacto
  </button>

  <!-- ★ NUEVO BOTÓN -->
  <button class="tab-btn"
          data-tab="formularioRegistro">
    Formulario de Registro
  </button>
</nav>
⚠️ El valor de data-tab="formularioRegistro" debe coincidir exactamente con el id de la sección que crearemos en el siguiente paso. Si no coinciden, JavaScript no podrá encontrar qué mostrar.

Parte C — Agregar la sección con <iframe>

Debajo de las secciones existentes (#registro y #contacto), agregamos una nueva <section> con un <iframe> que carga el archivo externo.

HTML — index.html (nueva sección) 
<!-- ④ FORMULARIO EXTERNO: cargado desde otro archivo -->
<section id="formularioRegistro" class="form-panel">
  <h2>Formulario de Registro</h2>
  <iframe
    src="formularioRegistro.html"
    width="100%"
    height="600"
    style="border: none; border-radius: 8px;"
  ></iframe>
</section>

<iframe> — ¿Qué es?

Es una etiqueta HTML que incrusta otra página web dentro de la página actual. Funciona como una "ventana" que muestra el contenido de otro archivo HTML.

src — La conexión

El atributo src="formularioRegistro.html" es la ruta al archivo. Como ambos archivos están en la misma carpeta, solo usamos el nombre.

Propiedad del iframeDescripción
srcRuta al archivo HTML que se va a cargar (la conexión entre archivos)
widthAncho del iframe (usamos "100%" para que ocupe todo el espacio)
heightAlto en píxeles del iframe
style="border: none"Quitamos el borde por defecto para que se vea integrado

Parte D — El JavaScript ya funciona

El código de app.js que ya tenemos no necesita ningún cambio. Recordemos cómo funciona:

JavaScript — app.js (sin cambios) 
// Selecciona TODOS los botones y TODOS los paneles
const tabButtons = document.querySelectorAll('.tab-btn');
const formPanels = document.querySelectorAll('.form-panel');

tabButtons.forEach(function(btn) {
  btn.addEventListener('click', function() {
    const target = btn.getAttribute('data-tab');

    // Quita .active de todos
    tabButtons.forEach(function(b) { b.classList.remove('active'); });
    formPanels.forEach(function(p) { p.classList.remove('active'); });

    // Activa solo el seleccionado
    btn.classList.add('active');
    document.getElementById(target).classList.add('active');
  });
});

¿Por qué funciona sin cambios? Porque el patrón limpiar todo → activar lo seleccionado es genérico:

  • querySelectorAll('.tab-btn') selecciona automáticamente los 3 botones (incluyendo el nuevo).
  • querySelectorAll('.form-panel') selecciona las 3 secciones (incluyendo la nueva).
  • getElementById(target) busca el id que coincide con el data-tab del botón clickeado.
🔑 Resumen de la conexión: El botón tiene data-tab="formularioRegistro" → JavaScript lee ese valor → busca getElementById("formularioRegistro") → encuentra la sección → le agrega la clase .active → el CSS muestra esa sección → el <iframe> dentro carga el archivo formularioRegistro.html.
02

Tipos de campos de formulario

HTML ofrece distintos tipos de <input> y otros elementos para capturar datos del usuario. Cada tipo tiene propiedades específicas que controlan su comportamiento.

▶ input type="text"

Campo de texto libre. Es el tipo más básico y común.

HTML 
<label>Nombre completo</label>
<input type="text"
       placeholder="Ej: Juan Pérez"
       maxlength="50"
       required
       value=""
       disabled
       readonly/>
PropiedadDescripción
placeholderTexto gris que aparece cuando el campo está vacío (pista visual)
maxlengthNúmero máximo de caracteres permitidos
requiredEl campo es obligatorio — el formulario no se envía si está vacío
valueValor inicial del campo
disabledEl campo aparece gris y NO se puede editar ni enviar
readonlyEl campo se puede leer pero NO editar (sí se envía)

▶ input type="number"

Solo acepta valores numéricos. Muestra flechas para incrementar/decrementar.

HTML 
<label>Edad</label>
<input type="number"
       placeholder="Ej: 25"
       min="1"
       max="120"
       step="1"/>
PropiedadDescripción
minValor mínimo permitido
maxValor máximo permitido
stepIncremento de cada paso (ej: 0.01 para decimales, 1 para enteros)

▶ input type="date"

Muestra un selector de fecha nativo del navegador.

HTML 
<label>Fecha de nacimiento</label>
<input type="date"
       min="1950-01-01"
       max="2010-12-31"
       value="2000-01-15"/>
PropiedadDescripción
minFecha mínima seleccionable (formato AAAA-MM-DD)
maxFecha máxima seleccionable
valueFecha seleccionada por defecto

▶ input type="email"

Valida automáticamente que el texto tenga formato de correo electrónico.

HTML 
<label>Correo electrónico</label>
<input type="email"
       placeholder="tu@email.com"
       required/>
💡 El navegador valida automáticamente que el texto contenga @ y un dominio. Si no cumple el formato, el formulario no se envía.

▶ input type="password"

Oculta los caracteres escritos con puntos o asteriscos.

HTML 
<label>Contraseña</label>
<input type="password"
       placeholder="Mínimo 6 caracteres"
       minlength="6"
       maxlength="20"
       required/>
PropiedadDescripción
minlengthCantidad mínima de caracteres requeridos
maxlengthCantidad máxima de caracteres permitidos

▶ textarea

Campo de texto multilínea, ideal para mensajes largos o comentarios.

HTML 
<label>Mensaje</label>
<textarea rows="4"
          cols="50"
          placeholder="Escribe tu mensaje..."
          maxlength="500"></textarea>
PropiedadDescripción
rowsNúmero de líneas visibles (alto del campo)
colsNúmero de caracteres de ancho visible
placeholderTexto de ayuda cuando está vacío
maxlengthLímite máximo de caracteres

▶ select

Lista desplegable para elegir una opción entre varias.

HTML 
<label>Género</label>
<select required>
  <!-- Opción vacía como placeholder -->
  <option value="" disabled selected>Selecciona...</option>
  <option value="F">Femenino</option>
  <option value="M">Masculino</option>
  <option value="O">Otro</option>
</select>
PropiedadDescripción
valueEl valor que se envía al servidor cuando se elige esa opción
selectedMarca la opción seleccionada por defecto
disabledLa opción aparece pero no se puede elegir (útil para placeholders)
🔑 La primera <option> con value="", disabled y selected funciona como un placeholder: el usuario ve "Selecciona..." pero debe elegir una opción real.
03

Validaciones con JavaScript

Aunque HTML tiene validaciones nativas (required, minlength, etc.), JavaScript permite crear validaciones personalizadas con mensajes de error claros.

  • Capturamos el formulario con getElementById.
  • Escuchamos el evento submit.
  • Usamos preventDefault() para evitar que la página se recargue.
  • Validamos cada campo y mostramos errores en el DOM.
JavaScript — validacion.js 
// ① Seleccionamos el formulario
const form = document.getElementById('miFormulario');

// ② Escuchamos el evento "submit"
form.addEventListener('submit', function(event) {

  // ③ Evitamos que la página se recargue
  event.preventDefault();

  // ④ Obtenemos los valores de los campos
  var nombre = document.getElementById('nombre').value;
  var email  = document.getElementById('email').value;
  var edad   = document.getElementById('edad').value;

  // ⑤ Limpiamos errores anteriores
  document.getElementById('errores').innerHTML = '';

  var mensajes = [];

  // ⑥ Validar nombre (no vacío, mínimo 3 caracteres)
  if (nombre === '') {
    mensajes.push('El nombre es obligatorio');
  } else if (nombre.length < 3) {
    mensajes.push('El nombre debe tener al menos 3 caracteres');
  }

  // ⑦ Validar email (contiene @)
  if (email === '') {
    mensajes.push('El email es obligatorio');
  } else if (email.indexOf('@') === -1) {
    mensajes.push('El email debe contener @');
  }

  // ⑧ Validar edad (entre 1 y 120)
  if (edad === '') {
    mensajes.push('La edad es obligatoria');
  } else if (edad < 1 || edad > 120) {
    mensajes.push('La edad debe estar entre 1 y 120');
  }

  // ⑨ Si hay errores, los mostramos
  if (mensajes.length > 0) {
    var listaHTML = '';
    for (var i = 0; i < mensajes.length; i++) {
      listaHTML += '<p>' + mensajes[i] + '</p>';
    }
    document.getElementById('errores').innerHTML = listaHTML;
  } else {
    // ⑩ Todo válido: mostramos éxito
    alert('Formulario enviado correctamente');
  }

});

El HTML del formulario necesita id en cada campo y un contenedor para los errores:

HTML 
<form id="miFormulario">
  <label>Nombre</label>
  <input type="text" id="nombre" placeholder="Tu nombre"/>

  <label>Email</label>
  <input type="email" id="email" placeholder="tu@email.com"/>

  <label>Edad</label>
  <input type="number" id="edad" placeholder="Tu edad"/>

  <!-- Contenedor para errores -->
  <div id="errores" style="color: red;"></div>

  <button type="submit">Enviar</button>
</form>
🔑 event.preventDefault() es esencial. Sin él, el navegador recarga la página al enviar el formulario y nunca verás tus validaciones.
04

Acciones en botones

HTML tiene 3 tipos de botones dentro de un formulario. Cada uno tiene un comportamiento diferente.

type="submit"

Envía el formulario. Dispara el evento submit y las validaciones.

type="reset"

Limpia todos los campos del formulario a sus valores iniciales.

type="button"

No hace nada por defecto. Perfecto para acciones personalizadas con JavaScript.

Sin type

Por defecto actúa como submit si está dentro de un <form>.

HTML 
<!-- Envía el formulario -->
<button type="submit">Enviar</button>

<!-- Limpia todos los campos -->
<button type="reset">Limpiar formulario</button>

<!-- Acción personalizada con JS -->
<button type="button" id="btnVer">Ver/Ocultar contraseña</button>

Ejemplo: un botón que muestra u oculta la contraseña:

JavaScript 
// Botón para mostrar/ocultar contraseña
var btnVer = document.getElementById('btnVer');
var passField = document.getElementById('password');

btnVer.addEventListener('click', function() {
  // Si el tipo actual es "password", lo cambiamos a "text"
  if (passField.type === 'password') {
    passField.type = 'text';
    btnVer.textContent = 'Ocultar contraseña';
  } else {
    passField.type = 'password';
    btnVer.textContent = 'Ver contraseña';
  }
});
⚠️ Usa type="button" cuando quieras que un botón haga algo con JavaScript sin enviar ni limpiar el formulario. Si olvidas el type dentro de un <form>, el botón actuará como submit.

▶ Parte JavaScript — validacion.js

Este es el archivo JavaScript completo con todas las validaciones y la acción del botón ver/ocultar contraseña.

JavaScript — validacion.js 
// ═══════════════════════════════════════
// VALIDACIÓN DEL FORMULARIO DE REGISTRO
// ═══════════════════════════════════════

// ① Seleccionamos el formulario y el contenedor de errores
var form = document.getElementById('miFormulario');
var erroresDiv = document.getElementById('errores');

// ② Escuchamos el evento "submit" del formulario
form.addEventListener('submit', function(event) {

  // ③ Evitamos que la página se recargue
  event.preventDefault();

  // ④ Obtenemos el valor de cada campo
  var nombre   = document.getElementById('nombre').value;
  var email    = document.getElementById('email').value;
  var edad     = document.getElementById('edad').value;
  var fecha    = document.getElementById('fecha').value;
  var genero   = document.getElementById('genero').value;
  var password = document.getElementById('password').value;

  // ⑤ Limpiamos errores anteriores
  erroresDiv.innerHTML = '';

  // Quitamos el borde rojo de todos los campos
  var campos = form.querySelectorAll('input, select, textarea');
  for (var i = 0; i < campos.length; i++) {
    campos[i].style.borderColor = '#e0e0e0';
  }

  var mensajes = [];

  // ⑥ Validar nombre (obligatorio, mínimo 3 caracteres)
  if (nombre === '') {
    mensajes.push('El nombre es obligatorio');
    document.getElementById('nombre').style.borderColor = 'red';
  } else if (nombre.length < 3) {
    mensajes.push('El nombre debe tener al menos 3 caracteres');
    document.getElementById('nombre').style.borderColor = 'red';
  }

  // ⑦ Validar email (obligatorio, debe contener @)
  if (email === '') {
    mensajes.push('El email es obligatorio');
    document.getElementById('email').style.borderColor = 'red';
  } else if (email.indexOf('@') === -1) {
    mensajes.push('El email debe contener @');
    document.getElementById('email').style.borderColor = 'red';
  }

  // ⑧ Validar edad (obligatorio, entre 1 y 120)
  if (edad === '') {
    mensajes.push('La edad es obligatoria');
    document.getElementById('edad').style.borderColor = 'red';
  } else if (edad < 1 || edad > 120) {
    mensajes.push('La edad debe estar entre 1 y 120');
    document.getElementById('edad').style.borderColor = 'red';
  }

  // ⑨ Validar fecha (obligatorio)
  if (fecha === '') {
    mensajes.push('La fecha de nacimiento es obligatoria');
    document.getElementById('fecha').style.borderColor = 'red';
  }

  // ⑩ Validar género (obligatorio)
  if (genero === '') {
    mensajes.push('Debes seleccionar un género');
    document.getElementById('genero').style.borderColor = 'red';
  }

  // ⑪ Validar contraseña (obligatorio, mínimo 6 caracteres)
  if (password === '') {
    mensajes.push('La contraseña es obligatoria');
    document.getElementById('password').style.borderColor = 'red';
  } else if (password.length < 6) {
    mensajes.push('La contraseña debe tener al menos 6 caracteres');
    document.getElementById('password').style.borderColor = 'red';
  }

  // ⑫ Mostrar errores o mensaje de éxito
  if (mensajes.length > 0) {
    var html = '';
    for (var j = 0; j < mensajes.length; j++) {
      html += '<p style="color:red;">' + mensajes[j] + '</p>';
    }
    erroresDiv.innerHTML = html;
  } else {
    alert('Formulario enviado correctamente');
  }

});

// ═══════════════════════════════════════
// BOTÓN VER / OCULTAR CONTRASEÑA
// ═══════════════════════════════════════

var btnVer = document.getElementById('btnVer');
var passField = document.getElementById('password');

btnVer.addEventListener('click', function() {
  if (passField.type === 'password') {
    passField.type = 'text';
    btnVer.textContent = 'Ocultar contraseña';
  } else {
    passField.type = 'password';
    btnVer.textContent = 'Ver contraseña';
  }
});

// ═══════════════════════════════════════
// LIMPIAR ERRORES AL PRESIONAR "RESET"
// ═══════════════════════════════════════

form.addEventListener('reset', function() {
  erroresDiv.innerHTML = '';
  var campos = form.querySelectorAll('input, select, textarea');
  for (var i = 0; i < campos.length; i++) {
    campos[i].style.borderColor = '#e0e0e0';
  }
});

06

Verificar la estructura de carpetas

Antes de abrir el proyecto en el navegador, asegurate de que todos los archivos estén en su lugar. Así debe quedar la estructura final del proyecto:

ESTRUCTURA FINAL DEL PROYECTO 
mi-proyecto/
│
├── index.html                ← Página principal (menú con tabs)
│
├── formularioRegistro.html   ← Formulario externo (se carga con iframe)
│
├── app.js                    ← Lógica de navegación entre tabs
│
├── validacion.js             ← Validaciones del formulario de registro
│
└── style/
    └── style.css             ← Estilos visuales del proyecto

Cada archivo tiene una responsabilidad clara:

index.html

Contiene el menú de navegación con 3 botones (Registro, Contacto, Formulario de Registro) y las 3 secciones correspondientes.

formularioRegistro.html

Página independiente con el formulario completo. Se carga dentro de index.html mediante un <iframe>.

app.js

Controla la navegación por tabs: escucha clicks en los botones del menú y muestra/oculta las secciones.

validacion.js

Controla las validaciones del formulario de registro: verifica campos, muestra errores y maneja el botón ver/ocultar contraseña.

style/style.css

Define la apariencia visual del proyecto: colores, tamaños, espaciados, bordes y animaciones.

¿Quién carga a quién?

index.html carga style.css y app.js.
formularioRegistro.html carga validacion.js.

Diagrama de conexiones entre archivos:

CONEXIONES 
index.html
├── carga CSS ──────→ style/style.css     (<link href="style/style.css">)
├── carga JS ───────→ app.js              (<script src="app.js">)
└── carga HTML ─────→ formularioRegistro.html  (<iframe src="...">)
                       │
                       └── carga JS ──→ validacion.js  (<script src="validacion.js">)
💡 Importante: Todos los archivos deben estar en las rutas correctas. Si mueves un archivo de carpeta, debes actualizar la ruta en el href o src correspondiente. Por ejemplo, si style.css está dentro de style/, la ruta debe ser "style/style.css", no solo "style.css".
