GUÍA PASO A PASO

Menú con 2 Formularios
en HTML, CSS y JS

Aprende a construir un sistema de navegación con pestañas que muestre distintos formularios de forma dinámica, sin librerías externas.

HTML5 CSS3 JavaScript ES6

Estructura del proyecto

Todo vive en un único archivo index.html. Lo dividiremos en 3 partes: estructura HTML, estilos CSS y lógica JS.

01

Crear la estructura HTML base

Define el esqueleto del documento. Necesitamos un contenedor principal, la barra de menú (con dos botones/tabs) y dos secciones de formulario ocultas por defecto.

  • El <nav> contendrá los botones del menú.
  • Cada <section> representa un formulario.
  • Usamos el atributo data-tab para vincular botones con secciones.
HTML 
<!-- index.html -->
<!DOCTYPE html>
<html lang="es">
<head>
  <meta charset="UTF-8"/>
  <meta name="viewport" content="width=device-width, initial-scale=1.0"/>
  <title>Menú con Formularios</title>
  <link rel="stylesheet" href="style.css"/>
</head>
<body>

  <div class="container">

    <!-- ① MENÚ DE NAVEGACIÓN -->
    <nav class="menu">
      <button class="tab-btn active"
              data-tab="registro">
        Registro
      </button>
      <button class="tab-btn"
              data-tab="contacto">
        Contacto
      </button>
    </nav>

    <!-- ② FORMULARIO 1: REGISTRO -->
    <section id="registro" class="form-panel active">
      <h2>Crear cuenta</h2>
      <form>
        <label>Nombre</label>
        <input type="text" placeholder="Tu nombre"/>

        <label>Email</label>
        <input type="email" placeholder="tu@email.com"/>

        <label>Contraseña</label>
        <input type="password" placeholder="••••••••"/>

        <button type="submit">Registrarse</button>
      </form>
    </section>

    <!-- ③ FORMULARIO 2: CONTACTO -->
    <section id="contacto" class="form-panel">
      <h2>Contáctanos</h2>
      <form>
        <label>Asunto</label>
        <input type="text" placeholder="¿En qué podemos ayudar?"/>

        <label>Mensaje</label>
        <textarea rows="4"
                  placeholder="Escribe tu mensaje aquí..."></textarea>

        <button type="submit">Enviar mensaje</button>
      </form>
    </section>

  </div>

  <script src="app.js"></script>
</body>
</html>
💡 El atributo data-tab="registro" en el botón debe coincidir exactamente con el id="registro" de la sección. Esto es la clave que usará JavaScript para saber qué mostrar.
02

Agregar los estilos CSS

El CSS maneja dos cosas fundamentales: la apariencia visual del menú y la visibilidad de los paneles. La clase .active es la protagonista.

  • Por defecto, todos los paneles están ocultos con display: none.
  • Solo el panel con la clase .active se muestra.
  • El botón activo recibe un estilo visual diferenciado.

03

Escribir la lógica JavaScript

El JS es el cerebro. Cuando el usuario hace click en un botón del menú, el script quita la clase .active de todos los elementos y la agrega solo al seleccionado.

  • Seleccionamos todos los botones y todos los paneles.
  • Iteramos con forEach para limpiar las clases activas.
  • Leemos el data-tab del botón clickeado para saber qué panel activar.
JavaScript — app.js 
// ① Seleccionamos todos los botones y paneles
const tabButtons = document.querySelectorAll('.tab-btn');
const formPanels = document.querySelectorAll('.form-panel');

// ② Escuchamos clicks en cada botón
tabButtons.forEach(function(btn) {

  btn.addEventListener('click', function() {

    // A) Leemos qué tab queremos mostrar
    const target = btn.getAttribute('data-tab');

    // B) Quitamos .active de TODOS los botones
    tabButtons.forEach(function(b) {
      b.classList.remove('active');
    });

    // C) Quitamos .active de TODOS los paneles
    formPanels.forEach(function(panel) {
      panel.classList.remove('active');
    });

    // D) Activamos el botón clickeado
    btn.classList.add('active');

    // E) Activamos el panel cuyo id = data-tab
    document.getElementById(target)
            .classList.add('active');
  });

});
🔑 El flujo siempre es el mismo: limpiar todo → activar solo lo seleccionado. Este patrón funciona para 2, 3, 5 o cualquier cantidad de tabs sin cambiar el código.
04

Verificar la estructura de carpetas

Si separas los archivos, tu proyecto debe quedar así:

ESTRUCTURA 
mi-proyecto/
├── index.html    ← estructura y contenido
├── style.css     ← apariencia visual
└── app.js        ← lógica de las tabs

✅ Tiene .active

El panel o botón es visible y aparece destacado en el menú.

❌ Sin .active

El panel está oculto con display:none y el botón luce inactivo.
